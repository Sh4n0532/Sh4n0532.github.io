Git and Github

October 10, 2020

Installing Git

To install Git go to their official website https://git-scm.com. Go to Download and select your operating system, then just go through the normal installation process.

If you are using Mac and also familiar with Homebrew or you want try it out you can go to this page to see details about Homebrew. You can install Git with Homebrew by entering brew install git.

Setup Git

After installing Git you should first verity the installation and check the version installed with git --version. If it successfully show the version installed that means you have just installed Git correctly.

Before you start working with Git you should configure your name and email. This is important information to let people know who made the changes.

  • git config --global user.name "your_name" to set your username.
  • git config --global user.email "email_address" to set your email address.
  • git config --list to list out all the configuration that you have set.

If you need help with some command you can enter git help <command> to see more details about the command and the operation that command perform.

To Track or Untrack a Local Folder

To start tracking a folder you need a .git file inside the folder. Go to the folder you want to start tracking and enter git init. It will create a .git file. It is a hidden file so if you want to see the file you need to enter ls -al to list out all the files including the hidden one.

To stop tracking a folder, go to the folder you are currently tracking and simply remove the .git file from the directory. You can simply find the file and delete it or use the command line way to remove it with rm -rf .git.

File Status

Enter git status to view the status of the tracking folder.

If there are some files that you do not want to be track you can create a .gitignore file to make Git automatically ignore those files.

  • Create a .gitignore file with touch .gitignore.
  • Open .gitignore with any text editor you like.
  • Add the files name that you want Git to ignore into this file.
  • After you edit the file you can run git status again to check that the ignored files are no longer showing up.

Staging Area

To add files to staging area enter:

  • git add -A will add all files from the working directory to staging area
  • git add <filename> allow you to specify which file will be added to the staging area

To remove files from staging area:

  • git reset remove all the files from staging area
  • git reset <filename> to remove a particular file from staging area.

Commit Changes

After you have add all the files you want to commit to staging area, the files are ready to be commited.

  • git commit -m 'your_message' is the command used to commit changes locally.
  • git log will show the details of the commit you have just made.

Push Local Changes to Remote Repository on GitHub

After the changes is commited we need to push the updated version of our work to GitHub repository.

Before we push our changes we should first pull any changes that is done to the repository since the last time we pull with git pull origin masteer.

After our work is up-to-date with the repository one we can push the changes we made to the repository. Enter git push origin master to push the changes to the remote repository.

To Clone an Existing Remote Repository

If you want to work with an existing project you can clone the project from GitHub repository to your local machine. To clone a repository enter git clone <url_of_the_repository> <location_to_clone_to>

Branch

Sometimes you may need to create a new branch for a new feature to prevent it from breaking the old working code or for unit test.

To create a new branch, enter git branch <branch_name>. You can use git branch to view all your branches.

To start working on the new branch you have just created, you need to checkout the branch, git checkout <branch_name>.

To push your changes to the new branch all the process is same as normal push except this time you need to push to your new branch not master, git push -u origin <branch_name>.

Merge Branch

When you are happy with the work in your branch you can merge it with master.

To merge other branch with master:

  • You need to checkout master first by enter git checkout master.
  • As usual pull the changes that other people may have made to the repository.
  • git merge <branch_name> to merge the branch with master.
  • Then git push origin master to push the changes back to the remote repository.

Then the branch is merged with master. You can check all the branches that merge to master before by git branch --merged.

After merging the the branch we can now delete the branch.

  • git branch -d <branch_name> to delete the branch locally.
  • To delete the branch in the remote repository enter git push origin --delete <branch_name>
