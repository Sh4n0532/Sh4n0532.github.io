Two Sum

November 15, 2020

Problem Description

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

Example 1:

Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Output: Because nums[0] + nums[1] == 9, we return [0, 1].

Solutions

class Solution 
{
    public int[] twoSum(int[] nums, int target) 
    {
        int total = 0;
        int[] result = new int[2];
        
        for (int i = 0; i < nums.length; i++)
        {
            for (int j = i + 1; j < nums.length; j++)
            {                
                total = nums[i] + nums[j];
                if (total == target)
                {
                    result[0] = i;
                    result[1] = j;
                    break;
                }
            }
        }
        return result;
    }
}
  • This is a brute force solution to this problem.
  • We iterate through the list twice and try to add them up and compare it with the target we want.
  • The inner loop is i + 1 because we don’t want to use same element and it also prevent duplicate checking.
  • If they add up to the target we store their index number into a new array then return the array.
  • Time complexity: O(N^2)
class Solution {
    public int[] twoSum(int[] nums, int target) 
    {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i< nums.length; i++)
        {
            map.put(nums[i], i); //key value pair
        }
        for (int i = 0; i < nums.length; i++)
        {
            int diff = target - nums[i];
            if (map.containsKey(diff) && map.get(diff) != i)
            {
                return new int[] {map.get(diff), i};
            }
        }
        throw new IllegalArgumentException("No match");
    }
}
  • Solution with HashMap.
  • Create a map and store all the elements from the list into the map with their index number as key value pair. Data as key and index number as value because we want to get the index number later.
  • Then we loop through the list again, this time for finding the match pair.
  • int diff = target - nums[i]; we want to find the index of diff if diff exist in the list.
  • With map we can see if it contains key diff and if yes we should also check if its value is similar to i because we don’t want to use an element twice.
  • If the key is in the map and its value is not equal to i then it is the element we want to find.
  • With this method the time complexity is O(N) but it is not very good in space complexity.
class Solution {
    public int[] twoSum(int[] nums, int target) 
    {
        Map<Integer, Integer> map = new HashMap<>();
        
        for (int i = 0; i < nums.length; i++)
        {
            int diff = target - nums[i];
            if (map.containsKey(diff) && map.get(diff) != i)
            {
                return new int[] {map.get(diff), i};
            }
            else
            {
                map.put(nums[i], i);
            }
        }
        throw new IllegalArgumentException("No match");
    }
}
  • This is also a HashMap solution but it is one pass. You only loop through the list once.
  • When looping through the list calculate the diff and check if it is in the map at the same time.
  • If it is in the map and its value is not equal to i then it is the one we want.
  • If it is not in the map we add the current i element into the map.
  • The differences between this solution with the previous one is that in the previous one we add all the element into the map first then only we compare. This method we add element and compare at the same time so we only need to loop through the list once.
