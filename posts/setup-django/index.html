Setup Django

October 13, 2020

Before you start to work with Django you must have Python installed. You can check if you have python with python --version or python3 --version in your terminal. If the command successfully return your python version then you are good to go.

To install Python, please go to Python to see the complete process.

Virtual Environment

Normally we will not install Django globally on a machine. We will create a virtual environment and install Django only in the environment. Virtual environment is a separated space that you can work with that any changes in it will not affect your other projects. This way you can work with different version of Django with different project if you need.

  • To create a virtual environment, first create a new empty directory. You can simply create a new file the normal “New Folder” way or using terminal with mkdir folder_name. Then cd into the folder.
  • Enter python3 -m venv env_name to create a new virtual environment.
  • After creating the environment you need to activate the environment to start working in it. Enter source env_name/bin/activate. When the virtual environment is activated the environment name will appear in a bracket.
  • Now you are inside the virtual environment, you can start installing django with pip install django.
  • To check if django is successfully installed, enter django-admin --version.
  • To exit the virtual environment, enter deactivate.

Django Project

  • To start a new django project to work with, enter django-admin startproject project_name.
  • After enter the command this is what will be created.

  • To run your project, navigate to the file containing manage.py and enter python manage.py runserver.
  • Your website will be started locally in localhost with a link provided. Copy the link and paste it into your browser to see the output.
