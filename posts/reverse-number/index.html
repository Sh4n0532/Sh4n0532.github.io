Reverse Number

November 22, 2020

Problem Description

Given a 32-bit signed integer, reverse digits of an integer.

Note: Assume we are dealing with an environment that could only store integers within the 32-bit signed integer range: [−2^31, 2^31 − 1]. For the purpose of this problem, assume that your function returns 0 when the reversed integer overflows.

Input: x = 123
Output: 321

Solutions

class Solution 
{
    public int reverse(int x) 
    {   
        long rev = 0;
        while (x != 0)
        {
            int temp = x % 10;
            x = x / 10;
            rev = rev * 10 + temp;
        }
        
        if (rev > Integer.MAX_VALUE || rev < Integer.MIN_VALUE)
        {
            return 0;
        }
        return (int)rev;
    }
}
  • Create a variable with long data type to store the reverse result because it has greater range than int data type. We will check the range later.
  • When x is not equal to 0 we will continue to divide x and get its last digit with modulus. We cannot use x >= 1 because there are also negative value since this is signed integer.
  • When we want to add a new digit we multiple the current digit by 10 so their place value increase.
  • When we go out from the loop that means we already have the reversed integer stored in rev. Now we need to change its range to prevent overflow.
  • If it is overflow we simply return 0 else we convert it into int because the return type of the method is int.
class Solution 
{
    public int reverse(int x) 
    {
        String rev = new StringBuilder().append(Math.abs(x)).reverse().toString();
        try
        {
            if (x < 0)
                return Integer.parseInt(rev) * -1;
            else
                return Integer.parseInt(rev);
        }
        catch (Exception e)
        {
            return 0;
        }
    }
}
  • With the help of Java built-in library and methods.
  • Take the absolute value of x and convert it to string and reverse it.
  • If x is smaller than 0 meaning that it is a negative value than we will convert it to Integer and multiply -1 into it so it will become negative.
  • If it does not work that means it is out of range so we need to catch the exception and return 0.
