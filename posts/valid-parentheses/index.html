Valid Parentheses

December 20, 2020

Problem Description

Given a string s containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid.

An input string is valid if:

  1. Open brackets must be closed by the same type of brackets.
  2. Open brackets must be closed in the correct order.

Example 4:

Input: s = "([)]" - the order must also be correct
Output: false

Solution

class Solution 
{
    public boolean isValid(String s) 
    {
        HashMap<Character, Character> map = new HashMap<>();
        map.put(')', '(');
        map.put(']', '[');
        map.put('}', '{');
        
        Stack<Character> stack = new Stack<>();
        for (int i = 0; i < s.length(); i++)
        {
            char current = s.charAt(i);
            if (map.containsKey(current)) //closing
            {
                if (stack.empty() || map.get(current) != stack.peek())
                    return false;
                else // if matching we pop the open bracket in stack
                    stack.pop(); 
            }
            else //opening
            {
                stack.push(current);
            }
        }
        return stack.empty();
    }
}

Explanation

  • First we create a map to store the parentheses pair. Closing as key, opening as value, because when we encounter an opening parentheses it is easy, we don’t need to do anything except pushing it into stack to keep track of it.
  • We create a stack, this is for storing open parentheses.
  • Then we loop through the string by character, if we encounter an opening parentheses just push it into the stack.
  • If we get a closing parentheses we will check two things, one is if the stack is empty. If the stack is empty it is impossible to find any open parentheses to match it so return false.
  • Second thing is we will use this closing parentheses to get it value which is the opening pair from the map and check if it match with what is now at the top of the stack. If it doesn’t match return false.
  • If the stack is not empty and it also match with what is in the top of the stack then it means this is the closing pair so we pop the opening pair in the stack to remove it, meaning it is closed.
  • At the end, if the stack is clear and we already loop through all the characters in the string then this string is consider valid. So if at the end the stack is empty we will return true.
